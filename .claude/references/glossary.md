# 용어 사전

대화 및 코드에서 아래 용어를 일관되게 사용한다. 애매한 용어가 나오면 사용자에게 즉시 확인한다.

## IA 작업
| 용어 | 정의 |
|------|------|
| 원본 파일 | 거래처에서 메일/웹하드로 들어온 가공 이전 파일 (AI/EPS) |
| 출력 파일 | 디자이너 또는 IA가 가공한 인쇄 가능한 결과물 |
| 가공 | 원본 파일 → 출력 파일로 변환하는 전체 과정 (추출 + 후처리) |
| 추출 | 가공의 첫 단계. 원본에서 디자인 바운드를 잡아내는 것 |
| 후처리 | 가공의 두 번째 단계. 여백/재단선/돔보를 추가하는 것 |
| 바운드 | 디자인 영역의 크기 (가로×세로) |
| 클리핑 마스크 | 디자인 영역을 잘라내는 마스크 |
| 그룹 | AI 파일 안의 디자인 묶음 단위 |
| 블리드/여분 | 재단 여분 (상하좌우 mm) |
| 재단선 | 후가공을 위해 추가한 여백을 구분하는 선 |
| 돔보 | 펀칭 구멍 위치를 표시하는 마크 |

## IA 학습 (프로젝트)
| 용어 | 정의 |
|------|------|
| IA 학습 | 원본→출력 자동 가공을 위한 학습 프로젝트 전체 |
| 학습 데이터 | 가공 패턴 학습용 데이터 전체 |
| 사례 | 학습 데이터의 개별 레코드 1건 (원본+주문정보+가공결과 묶음) |
| 분석 데이터 | AI 파일 내부 구조 추출 결과 전체 |
| JSX 데이터 | JSX 스크립트로 추출한 AI 파일 내부 구조 (아트보드, 레이어, 클리핑마스크, 오브젝트 좌표 등) |
| 시각 데이터 | Vision API로 원본↔출력을 시각적으로 비교 분석한 결과 |
| 재단 데이터 | DXF 파일에서 파싱한 재단 좌표/경로 데이터 |

## MES 업무
| 용어 | 정의 |
|------|------|
| 주문 | 인쇄 요청, 상품 주문, 샘플 요청 등 모두 포함 |
| 품목 | 현수막, 솔벤시트 등 현장 기본 분류 단위 |
| 라인 | 주문서에서 하나씩 추가되는 개별 항목 (품목 + 규격 + 수량) |
| 거래처 | 주문을 넣는 업체 |
| 카드 | 생산 칸반보드의 작업 단위 |
| 확정 | 디자이너가 디자인 완료 처리 |
| 미수금 | 미회수 금액 |

> 이 사전은 점진적으로 업데이트한다. 새 용어가 등장하면 사용자에게 정의를 확인한 후 추가.
