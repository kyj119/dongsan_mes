# Brainstorming

신규 기능이나 구조 변경 시, 코드 작성 전에 아래 단계를 반드시 따른다.

## 1. 의도 파악
- "왜 이 요청을 했는가?" → 비즈니스 맥락에서 **진짜 목적**
- 누가 쓰는 기능인가? (디자이너/오퍼레이터/경리/관리자)
- 미수금·생산 흐름·기존 데이터에 영향이 있는가?
- 숨겨진 전제나 연쇄 영향은?

## 2. 요구사항 정리
"제가 이해한 바" 형태로 bullet 3~5개 요약 후 사용자 확인 요청.
모호한 점이 1개라도 있으면 **즉시 질문** (가/나/다 선택지 형태).

## 3. 설계 선택지
- 2~3가지 접근법 제시 (장단점 포함)
- 기존 코드 패턴과의 일관성 확인 (Explore 에이전트 활용)
- 영향 범위 명시 (파일, API, DB 테이블)
- 대형 기능(5 Phase 이상)은 세션 분리 제안

## 4. 합의 후 착수
- 사용자가 선택지를 확정한 후에만 구현 시작
- 구현 계획은 Phase별로 정리 후 제시
