## 시스템 점검 & 운영 모니터링

요청 수준에 따라 **개발 환경 점검** 또는 **운영 모니터링**을 자동 선택.

### 이 스킬의 범위 (pm-report와의 경계)

| 구분 | monitor | pm-report |
|------|---------|-----------|
| **초점** | 런타임 상태 (지금 돌고 있는 것) | 코드/작업 현황 (지금까지 한 것) |
| **대상** | workerd 프로세스, D1 DB, NAS, IA 로그, 하트비트 | git 커밋, 파일 변경, 마이그레이션 번호, plans/ 기획 |
| **질문** | "서버 살아있어?" / "에러 쌓이고 있어?" | "어디까지 왔어?" / "뭘 놓쳤어?" |
| **사용 시점** | 개발 시작 전, 장애 의심 시, 배포 후 | 세션 시작/종료, 주간 보고, 체크포인트 |

둘 다 필요한 상황(예: 배포 직후 "이상 없나?" 확인)이면 `pm-report` → `monitor` 순서로 실행.

### 개발 환경 점검 (간단)

"health check", "환경 확인", "개발 시작 전 확인" 같은 요청 시 실행.

5가지 항목을 순서대로 점검:
1. **D1 DB 존재**: `.wrangler/state/v3/d1/` 디렉토리 확인
2. **마이그레이션**: 최신 번호 확인, 미적용 파일 탐지
3. **시드 데이터**: users, items, clients 테이블에 데이터 있는지 확인
4. **서버 포트**: 3000번 포트 사용 중인지 확인
5. **NAS 접근**: Z: 드라이브 접근 가능한지 확인

**출력**: 5항목 테이블 (✅/❌), 문제 있으면 해결 명령어 안내

### 운영 모니터링 (종합)

"모니터링", "시스템 점검", "에러 확인", "운영 상태" 같은 요청 시 실행.
**Fan-out → Fan-in 패턴** (CLAUDE.md 패턴 A 참조).

#### Phase 1: 병렬 수집 (에이전트 4개 동시 실행)
**Agent 1 (haiku)**: 웹 서버 — workerd 프로세스 확인, API 헬스 체크
**Agent 2 (sonnet)**: IA 로그 — ia_diag/error/debug.log 최근 내용, 에러 패턴 분류
**Agent 3 (haiku)**: 에이전트 하트비트 — agent_heartbeats 테이블 조회, 10분+ 비활성 경고
**Agent 4 (sonnet)**: DB 정합성 — 주요 테이블 레코드 수, 카드 미생성 주문 탐지

#### Phase 2: 종합 (PM이 직접 또는 1개 sonnet 에이전트)
- 4개 결과를 "시스템 상태" 테이블 1개로 병합
- 🔴(장애) / 🟡(경고) / 🟢(정상) 3단계로 심각도 환산
- 권장 조치는 장애·경고가 있을 때만 표시

**출력 형식**:
```markdown
# 운영 모니터링 (YYYY-MM-DD HH:MM)

## 시스템 상태
| 컴포넌트 | 상태 | 비고 |
|---------|------|------|
| 웹 서버 | 🟢/🔴 | 응답코드 |
| IllustratorAutomat | 🟢/🟡/🔴 | 최근 에러 N건 |
| LogWatcher | 🟢 N대 활성 | |
| DB | 🟢 | orders: N건 |

## 권장 조치
(문제 발견 시만 표시)
```
