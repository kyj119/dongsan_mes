## 프로젝트 상태 & PM 보고서

요청 수준에 따라 **간단 모드** 또는 **종합 모드**를 자동 선택한다.

### 이 스킬의 범위 (monitor와의 경계)

| 구분 | pm-report | monitor |
|------|-----------|---------|
| **초점** | 코드/작업 현황 (지금까지 한 것) | 런타임 상태 (지금 돌고 있는 것) |
| **대상** | git 커밋, 파일 변경, 마이그레이션 번호, plans/ 기획 | workerd 프로세스, D1 DB, NAS, IA 로그, 하트비트 |
| **질문** | "어디까지 왔어?" / "뭘 놓쳤어?" | "서버 살아있어?" / "에러 쌓이고 있어?" |
| **사용 시점** | 세션 시작/종료, 주간 보고, 체크포인트 | 개발 시작 전, 장애 의심 시, 배포 후 |

둘 다 필요한 상황이면 `pm-report` → `monitor` 순서로 실행.

### 간단 모드 (기본)

"상태 확인", "status", "지금 어디까지?" 같은 요청 시 실행.

```bash
git status --short
git log --oneline -5
```
- 최신 마이그레이션 번호 확인 (`migrations/*.sql` Glob)
- 최근 변경 파일 확인 (`git diff --name-only HEAD~3`)

**출력**: 테이블 1개로 요약 (브랜치, 미커밋, 최신 마이그레이션, 최근 커밋)

---

### 종합 모드

"PM 보고", "종합 보고서", "전체 현황 분석" 같은 요청 시 실행.
**Fan-out → Fan-in 패턴** (CLAUDE.md 패턴 A 참조).

#### Phase 1: 병렬 수집 (에이전트 4개 동시 실행)
**Agent 1 (haiku)**: Git & 코드 변경 — 최근 10커밋 요약, 변경 파일 패턴 분석
**Agent 2 (sonnet)**: 코드 품질 — 대형 파일(100KB+), TODO/FIXME, 빈 catch 블록
**Agent 3 (haiku)**: 문서 동기화 — CLAUDE.md/README.md 숫자 vs 실제 파일 비교
**Agent 4 (sonnet)**: DB & 기획 — 마이그레이션 상태 + plans/ 미착수 기획안 목록

#### Phase 2: 종합 (PM이 직접 병합)
- 4개 결과를 "프로젝트 요약 + 코드 품질 경고 + 다음 권장 작업" 3섹션으로 구조화
- 중복 내용 제거, 우선순위순 재배열
- 사용자가 바로 액션 가능한 형태(실행 명령 + 파일 경로)로 정리

**출력 형식**:
```markdown
# PM 보고서 (YYYY-MM-DD)

## 1. 프로젝트 요약
| 항목 | 값 |
|------|-----|
| 총 API 라우트 / 페이지 / 마이그레이션 | N / N / N |
| 미커밋 변경 | N개 파일 |
| 최근 활동 모듈 | (최근 5커밋 기반) |

## 2. 코드 품질 경고
(문제 있을 때만 표시)

## 3. 다음 권장 작업
(분석 기반 제안 3개)
```
