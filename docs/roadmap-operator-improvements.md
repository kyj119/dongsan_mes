## E. 칸반 안정성 + QR 스캔 확장

### 목표
1. 칸반 드래그앤드롭 안정성 검증
2. QR 스캔 범위를 출고 외 상태 전환으로 확장

### 수정 대상
- `src/scripts/cards.js` — 드래그앤드롭 핸들러
- `src/scripts/cards.js` — `processQrScan()` 함수

### QR 스캔 확장 방안
```
현재: 스캔 → 출고(ship) 처리만
확장: 스캔 → 현재 상태에 따라 다음 상태로 전환
  PRINTING → PRINT_DONE (출력완료)
  PRINT_DONE → 출고 처리 (기존)
  HOLD → PRINTING (재개)
```

### 검증 방법
```
칸반 안정성:
1. /cards 접속 → 칸반 뷰
2. "진행중" → "출력완료" 카드 드래그 → 상태 변경 확인
3. "출력완료" → "보류" 드래그 → 보류 사유 모달 표시 확인
4. 페이지 새로고침 → 변경된 상태 유지 확인
5. 모바일 뷰에서 터치 드래그 동작 확인

QR 스캔 확장:
6. PRINTING 상태 카드번호 스캔 → "출력완료로 변경?" 확인
7. PRINT_DONE 상태 카드번호 스캔 → 출고 처리 확인
8. 존재하지 않는 카드번호 스캔 → 에러 토스트 확인
```

### 의존성
- 칸반 안정성은 독립적으로 착수 가능
- QR 확장은 현장 워크플로우 확인 후 결정
