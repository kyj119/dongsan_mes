## Part 1: 실사용 병목 (MES+ERP 구조적 문제)

### 🔴 즉시 체감되는 문제

#### 1. 주문 수정 후 카드 미동기 (가장 심각)
- **상황**: MANAGER가 납기를 5/10 → 5/8로 변경 → 카드는 여전히 5/10 표시
- **원인**: `canRegenerateCards` 조건 — 카드가 이미 RIP 진행 중이면 재생성 안 함
- **영향**: 현장 납기 혼란, 수동 카드 재생성 필요
- **수정안**: 카드 재생성 불가 시에도 `delivery_date`, `notes` 등 메타데이터는 동기화

#### 2. 출고 프로세스 4단계 (시간 낭비)
- **현재**: 배송사 선택 → 라벨 출력 → 선택 발송 → 출고 확정
- **문제**: 라벨 인쇄 후 따로 "출고 확정" 클릭 → 잊으면 시스템상 미출고
- **수정안**: "일괄 출고+라벨 인쇄" 1버튼으로 통합

#### 3. 대시보드 D-Day 부재
- **현재**: "납기 도래/지연" 패널은 있지만 KPI 상단에 숫자 없음
- **문제**: MANAGER가 아침에 "오늘 몇 건 출고?" 한눈에 못 봄
- **수정안**: 대시보드 KPI에 "오늘 출고 예정 N건" 카드 추가

### 🟡 중기 개선 (효율성)

#### 4. 메뉴 44개 과다
- OPERATOR는 4~5개만 필요 (현장카드, 후가공, 입고, 출고)
- **수정안**: 역할별 기본 메뉴 축약 or 즐겨찾기 상단 고정

#### 5. 반복주문 비효율
- 매주 동일 거래처에 동일 규격 주문 → 매번 처음부터 입력
- **수정안**: "주문 템플릿" 기능 (이미 `order_templates` 테이블 존재하나 UI 미연결)

#### 6. 주문서 필수/선택 구분 불명확
- "배송처", "출고방법"이 사실상 필수지만 선택으로 표시
- **수정안**: 필수 필드 `*` 표시 + 빈 제출 시 경고
