## Q1. 마감방식 프리셋 vs 드롭다운

### 현재 구조
주문서에 3가지 UI가 공존:
- **프리셋 버튼** (사방 열재단, 사방 접어미싱 등) — 1클릭 적용
- **사방 동일 드롭다운** — 하나 선택하면 4변 동일 적용
- **개별설정** (상/하/좌/우 따로 선택)

### 판단
- 프리셋 = 사방 동일 드롭다운과 기능 중복
- **프리셋 + 개별설정** 2가지만 유지 권장
- 사방 동일 드롭다운은 제거하고 프리셋에 통합

### 착수 시 수정 범위
- `src/scripts/orderForm.js` — 사방 동일 셀렉트 제거, 프리셋 UI 강화
- 예상 소요: 0.5일

## Q2. AI 파일 있는 라인 + 없는 라인 혼합

### 코드 분석 결과

**카드 생성 (MES)**: `getCardGroup()`은 ai_file_path를 고려하지 않음 → AI 파일 없는 품목도 동일 카드에 포함됨. **이건 정상** — 카드는 생산 단위, AI 파일 유무와 무관.

**IA 처리**: 
- 주문 레벨 `ai_file_path`가 null이면 ProcessOrderAsync 자체를 건너뜀 → **OK**
- 문제: 품목별로 `ai_file_path`가 있는 것과 없는 것이 섞이면, 없는 품목도 `ProcessItemAsync`로 전달됨
- `ProcessItemAsync`에서 빈 source 파일 검증 없이 JSX 실행 시도 → **오류 가능**

### 수정 필요
- `Program.cs` ProcessItemAsync 시작부에 `if (string.IsNullOrEmpty(aiFilePath)) return;` 가드 추가
- 예상 소요: 10분

## Q3. 여백 추가 시 재단선 미생성

### 원인 분석

재단선은 **IA(ProcessOrderItem.jsx)**에서만 생성됨. IA는 AI/EPS 파일이 있는 주문만 처리.

**테스트 시나리오별**:
- AI 파일 없는 수동 주문 → IA 미처리 → **재단선 생성 불가** (정상 동작)
- AI 파일 있는 주문 → IA 처리 → finishing 전달 → 재단선 생성 **되어야 함**

**데이터 전달 경로 확인**:
1. `order_items.finishing` 저장 ✅ (core.ts 708행)
2. `cards.finishing` 복사 ✅ (core.ts 985행)
3. GET /api/orders/:id에서 `oi.*` → finishing 포함 ✅
4. Program.cs에서 finishing 파싱 → finishingConfig 생성 ✅
5. ia_params.json에 finishing 전달 ✅
6. ProcessOrderItem.jsx에서 foldlines 그리기 ✅

**결론**: AI 파일 있는 주문으로 테스트하면 재단선 생성됨. 수동 주문에서는 불가.

### 향후 과제
- AI 파일 없는 주문에서도 마감 재단선이 필요하면, **카드 상세에서 직접 EPS 재단선 생성** 기능이 별도로 필요

## Q4. IA 콘솔에 "마감" 텍스트 없음

### 원인
Q3과 동일. AI 파일이 없는 주문은 IA가 ProcessOrderAsync를 건너뜀 → `✂️ 마감:` 로그 출력 안 됨.

AI 파일이 있는 주문에서 finishing이 설정되면 콘솔에 표시됨:
```
✂️ 마감: 상=접어미싱(4cm) 하=접어미싱(4cm) 좌=열재단(0cm) 우=열재단(0cm)
```

## Q5. 품목별 다른 마감방식 — 버그 확인

### 발견된 버그

**시나리오**: 카드 내 1번 품목=열재단(0cm), 2번 품목=사방접어미싱(4cm)

| 항목 | 현재 동작 | 문제 |
|------|-----------|------|
| final_width/height | `Math.max(0, 4) = 4cm` 적용 | 열재단 품목도 4cm 여백 적용됨 |
| cards.finishing | 첫 번째 유효값("열재단") 저장 | DB에는 "열재단"인데 규격은 접어미싱 기준 |
| 카드 표시 | "마감: 열재단 사방" | 실제 final 크기와 불일치 |

### 근본 원인
카드 = 그룹 단위(여러 품목 묶음)인데, finishing은 **품목별** 속성. 이 불일치가 문제.

### 해결 방향 (2가지)
**A. 카드 finishing을 배열로 저장** (권장)
- `cards.finishing` = `[{item_id: 1, ...열재단}, {item_id: 2, ...접어미싱}]`
- final_width/height는 최대값 유지 (인쇄 크기는 가장 큰 품목 기준)
- 카드 표시 시 품목별로 다른 마감 표시

**B. 마감방식이 다르면 카드 분리** 
- getCardGroup에 finishing을 키로 추가 → 열재단 카드, 접어미싱 카드 분리
- 장점: 깔끔. 단점: 카드 수 증가

### 현실적 판단
실제로 같은 주문 내에서 품목별 마감이 다른 경우가 드물면 **A 방식** 적용, 빈번하면 **B 방식** 검토

## Q6. 카드 레벨 마감 표시 혼동

### 문제
현재: 카드에 `마감: 접어미싱 사방` → 모든 품목이 같은 마감으로 오해

### 해결
카드 레벨이 아닌 **품목 라인별로 마감 표시** 이동:

```
카드 #01
├ 일반현수막 100×200  2EA  [코팅]  마감:접어미싱(4) 사방
├ 메쉬현수막 300×100  1EA         마감:열재단 사방
```

### 수정 범위
- `cards.js`의 `buildKanbanCard()` — 카드 레벨 마감 블록 제거
- 품목 라인 루프 내에서 `item.finishing` 또는 `order_items.finishing` 파싱하여 인라인 표시
- 칸반 카드의 품목 정보는 card_items JOIN order_items에서 가져오므로, **API에서 order_items.finishing을 포함**해야 함

## Q7. 작업지시서 그룹 품목 중복 표시

### 문제
그룹 품목(parent + children)이 있으면 포괄 라인과 하위 라인이 모두 표시됨.

### 해결
작업지시서에서 `parent_item_id`가 있는 품목은 하위 라인으로 인덴트하고, 부모만 있는 경우 부모만 표시.
또는 `parent_item_id`가 있는 품목(자식)만 표시하고 부모는 그룹 헤더로 처리.

### 수정 범위
- `printWorkOrder()` 함수에서 items 필터링 로직 추가
- 예상 소요: 30분

## Q9. 재고 카테고리 + 창고별 관리

### 현재 카테고리 기준
재고 페이지의 카테고리 = `items.category` (품목 대분류)

| items.category | 품목 수 | 예시 |
|----------------|---------|------|
| (빈 값) | 33 | category 미설정 품목 |
| 원단류 | 16 | PVC, 메쉬, 천 등 |
| 원자재 | 5 | 잉크, 솔벤트 등 |

**문제**: 빈 값 33건 → items 테이블에서 category가 비어있는 구매 품목들

### 창고별 관리
- `inventory.location` 필드 존재하지만 **미활용**
- `storage_zones` 테이블 존재, `/storage-zones` 페이지 존재
- **연결 안 됨**: 입고/출고 시 location 선택 UI 없음, 필터링 없음

### 개선 방향
1. items.category 빈 값 정리 (품목 관리에서 카테고리 일괄 설정)
2. 재고 목록에 `location` 필터 추가
3. 입고 시 storage_zone 선택 연동

## Q10. 재고 실사 완성도

### 현재 흐름
1. "새 실사 시작" → `is_purchase_item=1` 전체 품목 로드 (54건)
2. 품목별 실측 수량 입력
3. "제출" → 관리자 승인 대기
4. "승인" → 시스템 재고를 실측 수량으로 자동 조정

### 현실적 문제
- 54개 품목을 한번에 전부 세는 건 **비현실적**
- 카테고리/위치별 부분 실사 미지원
- 실사 중간 저장은 되지만 UX가 불편 (모든 품목이 한 목록에)
- 로스율 계산은 있지만 **품목별 로스 트렌드** 미제공

### 개선 제안
1. **부분 실사**: 카테고리 또는 위치 선택 → 해당 품목만 로드
2. **실사 주기 관리**: 품목별 마지막 실사일 표시, 미실사 경고
3. **로스 분석**: 품목별 실사 차이 이력 그래프
