# 작업지시서 활용 방향 리서치 결과

> **작성일**: 2026-05-02 | **용도**: 작업지시서 방향 결정을 위한 자료

---

## 1. 현수막/간판 업체의 실제 활용 패턴

### 물리적 운용
- **종이 작업지시서가 "작업과 함께 이동"**: 클립보드/바인더에 넣어 공정별 빈(bin)을 이동
- 대형 인쇄물: 말아놓은 현수막 위에 **클립으로 고정** 또는 투명 봉투에 동봉
- 라벨 부착은 비현실적 (현수막이 크고 말려있어서)

### 생성 시점
- **주문 확정(시안 승인) 시점**에 발행
- 이 시점에 작업번호(Job Number) 부여

### 크기/형태
- **A4 또는 A5** 가장 보편적
- NCR 복사지(2~3부): 고객용/생산용/사무용 분리하는 업체도 있음
- **상단에 큰 글씨로 Job Number** → 멀리서도 식별

### 수명
- 완료 후 6개월~1년 파일링 보관 (재주문/클레임 대응)
- 고객에게는 동봉하지 않음 (내부 문서)

## 2. 디지털 vs 물리적

| 방식 | 사용처 | 현실 |
|------|--------|------|
| PC/모니터 | 사무실 주문관리 | O |
| 종이 출력 | 생산현장 | **필수** — 작업자 대부분 종이 선호 |
| 태블릿 | 설치현장 | 선택적 |
| QR 스캔 | 공정 전환 기록 | 최근 추세 |

> "태블릿 도입을 시도해도 생산 현장 작업자들은 종이를 선호" — Signs101 커뮤니티 다수 의견

### 현실적 결론
**하이브리드**: 사무실=디지털(MES), 현장=종이 출력 + QR로 상태 전환

---

## 3. 현수막 업체 특수성

- 동시 다건 생산: 대형 프린터에서 여러 주문 연속 출력 → **출력 순서와 지시서 순서 매칭 핵심**
- 후가공 분기: 같은 출력물이라도 열재단/미싱/타공 등 다름 → **후가공 표기 필수**
- 설치 현장: 위치/담당자 정보 포함 시 설치팀에도 활용 가능

---

## 4. 실패 사례 / 주의점

| 실패 | 원인 | 대응 |
|------|------|------|
| 정보 과부하 | 모든 정보를 한 장에 → 안 읽음 | 공정별 필요 정보만 표시 |
| 디지털 전환 실패 | 현장에 태블릿 → 종이로 회귀 | 점진적 전환 |
| 번호 체계 혼란 | 비일관적 번호 부여 | 순차+날짜 조합 (현재 MES 방식 OK) |
| 부분 공정 추적 불가 | 단일 지시서로 여러 공정 | 디지털 상태 관리 병행 |

## 5. 동산현수막에 적용 가능한 방안

### 방안 A: 현재 MES + 종이 출력 (권장 1단계)

```
주문 확정 → MES에서 "작업지시서 인쇄" 클릭
    ↓
브라우저 인쇄 (window.print, 새 탭 아닌 인쇄 다이얼로그만)
    ↓
A4 종이에 출력 (현장 프린터)
    ↓
출력물과 함께 이동 (클립 고정)
    ↓
후가공 완료 → QR 스캔으로 MES 상태 전환
```

**포함 내용 (현장 작업자 기준)**:
- 큰 글씨 주문번호 + QR (스캔용)
- 거래처 + 납기 + 특이사항
- **품목별 카드**: 썸네일 + 규격 + 수량 + 후가공 + 마감 다이어그램
- 하단: 담당/확인 서명란

**제외 내용 (현장 불필요)**:
- 금액/단가 (사무 정보)
- 거래처 코드/연락처 (영업 정보)
- 디자인 파일명/경로 (기술 정보)

### 방안 B: 공정별 분리 지시서 (2단계)

```
1. 인쇄 지시서: 규격 + 수량 + 소재 + 색상참고
2. 후가공 지시서: 마감방식 + 후가공 상세 + 다이어그램
3. 출하 지시서: 납기 + 배송 방법 + 수량 확인
```

→ 공정별로 필요한 정보만 담아 정보 과부하 방지

### 방안 C: QR 중심 최소 라벨 (3단계)

```
출력물에 부착할 미니 라벨 (5x10cm):
┌───────────────────┐
│ [QR] 20260502-001 │
│ ABC광고 현수막 2EA│
│ 접어미싱 사방      │
└───────────────────┘
```

→ 상세 정보는 QR 스캔으로 MES에서 확인
→ 라벨 프린터(Brother QL-820NWB) 필요

## 6. 결정 필요 사항

1. **인쇄 형태**: 새 탭이 아닌 인쇄 다이얼로그만? → `iframe` + `window.print()` 방식
2. **이미지 포함 여부**: 썸네일 넣을지 (인쇄 시 잉크 소모 vs 확인 편의)
3. **공정별 분리**: 1장으로 통합 vs 인쇄/후가공/출하 분리
4. **QR 스캔 활용 범위**: 출고만? 공정 전환(인쇄완료→후가공→출하)?
5. **라벨 도입 여부**: 종이 지시서만? 미니 라벨도?
